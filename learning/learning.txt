1. 라이브러리 vs 프레임워크
공통점 : 누군가 미리 작성해 놓은 코드이고, 개발 속도를 더 빠르게 만들어주는 도구
차이점 : point word -> 제어, 누군가 만들어 놓은 규칙에 따라 작성하는 것을 준수 해야 되는 것이 프레임워크, 규칙을 바꿀 수도 없고 다른 프레임워크로 교체하는 것 또한 쉽지 않다. 그에 반대 되는 것이 라이브러리

2. 제이쿼리의 탄생
초반에 있었던 자바스크립트의 문제들(브라우저 호환성,잔 오류들 해결,깔끔한 코드작성)을 해결해주는 언어로 사용, 지금은 여러 버전을 거치면서 순수 자바스크립트만으로도 수행 할 수 있게 되어 제이쿼리의 목적이 사라졌다.
그러나 내가 취업하고 싶은 회사에서 제이쿼리를 사용한다면 배우는 정도로 활용해라. 그래도 JavaScript는 기본이다.
순수 자바스크립트를 배워야 하는 이유 : 1. 언어의 핵심 -> 다른 프레임워크나 라이브러리를 더 쉽게 배울 수 있음 2. 제이쿼리보다 더 빠르고 효율적 3. 최신 프레임워크들이 순수 자바스크립트 기반이므로

cf. 플러그인 : 기존 프로그램에 꽂아 넣어 새로운 기능을 추가하는 작은 프로그램. 필요한 기능만 골라서 추가할 수 있다는 것이 장점이다. 그래서 프로그램을 자신의 필요에 맞게 꾸밀 수도 있다. 
예를들어, 광고를 차단하는 플러그인을 설치하면 웹 서핑 할 때 귀찮은 광고들이 보이지 않게 됨. 또, 맞춤법 검사 플러그인을 추가하면 문서를 작성하면서 자동으로 맞춤법을 확인 할 수 있음. 

3. API(Application Programming Interface): API는 서로 다른 프로그램들이 소통하고 정보를 교환할 수 있게 해주는 도구.
작동 방식의 원리는 알 필요는 없음. 그저 API가 제공하는 기능의 결과만 가지면 됨. 
cf. 웹 API : 우리가 필요한 정보나 데이터를 가져 올 수 있게 해줌 ex. 날씨 정보, 주식 가격, 뉴스 기사
